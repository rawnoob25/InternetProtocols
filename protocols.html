Internet Protocols

DHCP

Purpose

To give a host on a network an IPv4 address

Procedure

  1. DHCP DISCOVER: Host sends a limited broadcast to either IPV4 address 255.255.255.255 or to MAC address FF-FF-FF-FF-FF-FF.
  2. DHCP OFFER: DHCP server on network (if present) responds to the broadcast and other hosts on network ignore the broadcast. The response includes a suggested IP address for the soon-to-be DHCP client. This response is the DHCP offer.
  3. DHCP REQUEST: DHCP client sends message to DHCP server accepting the IP address provided to it.
  4. DHCP ACK: DHCP server acknowledges receipt of the DHCP client's DHCP request.

Other Considerations

In large corporate networks there's usually PC that functions as a DHCP server. In smaller corporate networks and in home networks, a router usually functions as a DHCP server for hosts on the network, and it's in turn a DHCP client for an ISP's DHCP server.

Are DHCP requests from hosts on a home network sent to the router and then forwarded to the ISP's DHCP server? Is it the ISP's DHCP server that ultimately sends the host the DHCP ACK?

In situations where a host on a network doesn't have an IP address, it assigns itself an IP address whose first couple octets are 169.254; This is called an Automatic Private IP Address (APIPA); the host may lack an ip address due to any of the following:

  • It's been set up for manual IP configuration and was never given an IP address.
  • It has no associated DHCP server.
  • It has not sent a valid DHCP request to its DHCP server.

ARP

Purpose

When a device on the network wants to know another device's MAC address but does not, yet it knows the other device's IPv4 address, ARP allows the requesting device to obtain the MAC address of the other device. This is useful when a device needs to send a packet to another device on the local network (or needs to send the packet to the default gateway for transfer to another network), as the OSI layer 2 ethernet protocol cannot be completed without a valid destination MAC address.

Procedure

  1. ARP Request: Host 1 (seeking the MAC address of Host 2) sends a limited broadcast to 255.255.255.255 (encapsulated in ethernet frame with destination MAC address FF-FF-FF-FF-FF-FF) asking device with ip address matching that stated in encapsulated data field of ethernet frame to send it (Host 1) its (the device w/ matching ip address) MAC address.
  2. ARP Reply: Host 2 replies with its MAC address and all other hosts ignore the message.
  3. ARP cache update: Host 1 updates its ARP table (ARP table is a mapping on any device of IPV4 addresses to MAC addresses for every device in that network)
